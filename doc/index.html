Dispatch Bot

Automated dispatcher for Fuel Dude clients and pilots.

Features

Client information gathering

Bot asks client for information, and automatically interprets the information.

The information collected includes

  • CMDR name
  • current system
  • platform
  • life support status

Logging of missions, including Fuel Rats' lexicon

The bot generates a log file of every mission, and organizes them by mission number and client CMDR name.

Automatic mission progression

If the pilots use the appropriate lexicon, Dispatcher Bot will automatically send relevant information to the client and/or pilots.

Pilot statuses

Pilots have the ability to set themselves as on duty, reloading, and off duty using the appropriate commands. Dispatcher Bot uses this information to directly contact on duty pilots when one or more of the following happens:

  • No pilots accept the mission within 60 seconds
  • There are less than 2 pilots on a Code Red mission 60 seconds after the first pilot accepts the mission.

Paperwork-free missions

Because this bot is designed to be used in casual instances, paperwork is not an included part of the bot functionality. If you are looking for a professional environment to perform fuel rescue missions, please don't hesitate to check out the Fuel Rats' website.

Quick and Dirty Command List

  • !register
  • !unregister
  • !status on duty, or /status on, or /on
  • !status off duty, or /status off, or /off
  • !status reloading, or /reloading, or /rtb
  • !fr

Supported Lexicon

Lexicon is only interpreted by Dispatcher Bot in the #fueldudes channel.

  • bc+ - client's beacon visible in target system, confirms instancing
  • bc- - client's beacon not visible in target system
  • cr - used when the client loses power to life support
  • db+ - debriefing concluded
  • dc - pilot disconnected from the game
  • ez - client is in the exclusion zone
  • fr+ - friend request received
  • fr- - friend request not received
  • no instancing, or no instance - alias of bc-
  • #j - jumps remaining to client (example: 3j)
  • omw - "on my way"
  • Dudelert - pun on alert; signals all fuel rats that a client needs fuel
  • rtb - returning to base. will trigger the command !rtb
  • roger, or rgr - "Got it."
  • standby - pilot is available for missions. will trigger the command !on
  • standing down - pilot is self-unassigned from the mission
  • sys+ - pilot has arrived in the client's system
  • wr+ - wing request received
  • wr- - wing request not received. bot will remind user to send a friend request
